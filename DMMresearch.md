# DMMアフィリエイトAPI活用サイト設計まとめ

## DMM API v3 仕様

- **ベースURL**: `https://api.dmm.com/affiliate/v3/`
- **エンドポイント7種**: ItemList / FloorList / ActressSearch / GenreSearch / MakerSearch / SeriesSearch / AuthorSearch
- **認証**: `api_id` + `affiliate_id` をクエリに付けるだけ（署名不要、GET、JSON/XML対応）
- **ブラウザからのCORS不可** → Server Components か Route Handler でサーバープロキシ必須（API_IDを`NEXT_PUBLIC_`に絶対に付けない）
- **制限**: hits上限100件、offset上限50000、有効アフィID末尾は`-990`〜`-999`の10種のみ、レート制限値非公開（実運用は500ms〜1秒スリープ推奨）
- **主要レスポンスフィールド**: `title` / `affiliateURL` / `imageURL` (large/small/list) / `sampleImageURL` / `sampleMovieURL` (4サイズ) / `prices` / `review.average` + `review.count` / `iteminfo.actress` + `genre` + `maker` + `series`
- **FANZAリブランド**: 2018年8月以降 `DMM.R18` → `FANZA`。新規実装は`FANZA`を使う
- **next/image remotePatternsに登録必要な3ホスト**: `pics.dmm.co.jp` / `pics.dmm.com` / `awsimgsrc.dmm.co.jp`

## モバイルファースト設計原則

スマホでの体験を基準に設計し、PCは拡張として扱う。

### レイアウト・ナビゲーション

- **ボトムナビゲーション固定**（親指ゾーン最優先）: ホーム / 探す(スワイプ) / お気に入り / 検索 / マイページ の5タブ
- タップターゲットは最小44×44px（Apple HIG 基準）
- セーフエリア対応: `padding-bottom: env(safe-area-inset-bottom)` でホームインジケーターと重ならないようにする
- 高さは `vh` でなく `dvh`（Dynamic Viewport Height）を使用。モバイルブラウザのアドレスバー収縮に対応

### パフォーマンス

- **スケルトンUI** をすべての非同期コンポーネントに適用。コンテンツが入る前に枠だけ表示しLCPを下げる
- `next/image` の `blurDataURL` で画像のプレースホルダーをぼかし表示
- 動画（サンプル）は `IntersectionObserver` で画面内に入ったときだけ自動再生、離れたら停止
- **オプティミスティックUI**: お気に入りボタンはサーバー応答を待たず即座にUIを更新し、失敗時のみロールバック

### スワイプ・ジェスチャー

- スワイプフィード（形態E）は `touch-action: pan-y` でネイティブスクロールと共存させる
- 右スワイプ/左スワイプの判定は `pointerdown` → `pointermove` で X 方向の移動量が 50px を超えた時点で確定
- フィードバックは視覚（カードの傾き・色変化）＋ `navigator.vibrate(10)` の軽い振動（Android）

### PWA対応

- `manifest.json` でホーム画面への追加を促進（アイコン・テーマカラー・スプラッシュ）
- Service Worker で直近のランキング・お気に入りリストをオフラインキャッシュ
- プッシュ通知は Web Push API 経由（値下げアラート・新作通知）

## ターゲット設計：ライト・ヘビー両取り戦略

ライトユーザー（たまに探す・衝動買い）とヘビーユーザー（特定女優・ジャンルのコレクター）では求める体験が異なる。**入口はライト向けに広く取り、継続利用でヘビー向けに深くなる**設計が両立の鍵。

### ライトユーザー向け入口
| 施策 | 狙い |
|------|------|
| スワイプフィード（形態E） | 「何を見たいかわからない」層を発見体験で引き込む |
| 気分診断（3問→おすすめ） | 検索しなくても作品に出会える |
| セール・ランキング（形態A） | 「とりあえずお得なものを」層を拾う |
| OGP最適化 + X速報 | SNS経由の一見さんを獲得 |

### ヘビーユーザー向けリテンション
| 施策 | 狙い |
|------|------|
| お気に入り + 値下げ通知（形態B） | 「セール待ち」層を逃さず将来CVに変える |
| シリーズ完走トラッカー（形態F） | 継続訪問・まとめ買いの動機を作る |
| 女優ハブページ（形態D） | 「この女優の全作品を見たい」層のハブになる |
| 視聴・スワイプ履歴 → パーソナライズ | 訪問するたびに精度が上がる体験 |

**実装の優先順位**: 形態A（セール）→ 形態B（通知）→ 形態E（スワイプ）→ 形態D（ハブ）→ 形態F（トラッカー）

## 認証設計：ゲスト→ログインの段階的体験

### 認証方式

**名前 + メールアドレスのマジックリンク**（Supabase Auth）。パスワード不要で摩擦を最小化。

1. メールアドレスと表示名を入力
2. ワンタイムリンクがメールに届く
3. タップするだけでログイン完了（トークン保存、以降は自動ログイン）

### ゲストでできること（LocalStorage管理）

| 機能 | ゲスト | ログイン後 |
|------|--------|-----------|
| ランキング・セール閲覧 | ✅ | ✅ |
| スワイプフィード（履歴はセッション限り） | ✅ | ✅（DB保存・精度向上） |
| 気分診断 | ✅ | ✅ |
| アフィリンクへの遷移 | ✅ | ✅ |
| お気に入り保存（端末限定） | ✅ 5件まで | ✅ 無制限 |
| 値下げ・新作プッシュ通知 | ❌ | ✅ |
| パーソナライズ推薦 | ❌ | ✅ |
| シリーズ進捗トラッカー | ❌ | ✅ |
| ポイント・バッジ | ❌ | ✅ |
| 閲覧傾向グラフ（マイページ） | ❌ | ✅ |

### ログイン促進のタイミング（ゲートではなく誘導）

強制しない。自然な行動の延長で「ログインするとこうなる」を見せる。

- お気に入りを6件目に追加しようとした瞬間 → 「ログインすると無制限に保存できます」
- スワイプを10枚こなした後 → 「あなたの好みを学習中。ログインで続きを保存しますか？」
- 通知ベルをタップしたとき → 「ログインして値下げ通知を受け取る」
- 初回ログイン後、匿名セッションのLocalStorageデータを**そのままSupabaseに移行**（お気に入り・スワイプ履歴を引き継ぐ）

## ログイン後のUX向上施策

### 閲覧履歴・継続体験

- **プッシュ通知（Web Push）**: お気に入り作品の値下げ / 好きな女優の新作 / フォローしたシリーズの新刊
- **メール通知（週次）**: 「今週あなたへのおすすめ5選」を自動生成して送付。開封率の高いパーソナライズ内容にする

### ゲーミフィケーション（ポイント・バッジ）

## サイト形態

### 形態A: セール・ランキング特化（立ち上げ最優先）

ベントーグリッドUI（Apple風大小タイル混在）でセール作品・週間ランキング・新人作品を配置。
- 商品カード必須要素: 取消線定価 + 赤字割引 + %バッジ + 残時間カウントダウン + 星評価 → CVR 10〜30%改善
- FANZA同人90%OFFクーポンは最強CV要素なので常時バナー固定
- ランキング軸: 日次/週次/月次/ジャンル別/新人/検索急上昇の6軸
- 独自スコア: `★平均 × log(レビュー件数) × 新しさ係数` で自動ランキング生成

### 形態B: お気に入り → 値下げ・セール通知

- UI: ❤️ お気に入り / 🔔 値下げ通知 / 🔥 90%OFF通知
- 技術: Supabase + Push API + PWA + メール通知
- 未来CV・リピートCVを取るためSEO依存を減らせる

### 形態C: パーソナライズドプレイリスト・診断

- **プレイリスト**: 「女優名/ジャンル/作品URL」入力 → ハイブリッド推薦（協調フィルタ + コンテンツ類似ベクトル）で自動生成。Spotify風テーマ別提案（朝活/寝落ち/シリーズ完走）
- **診断型**: 「今日はどんな気分？」3問 → おすすめ作品表示。発見体験型でTikTok型と相性良い
- LocalStorage + Supabase匿名ユーザーでcookielessパーソナライズ
- プレイリスト内全作品にアフィURL埋め込み、まとめ買いボタンで高単価CV

### 形態D: 女優・作品ハブ（中長期SEO資産）

ActressSearch + ItemList で女優ページを自動生成（バスト/ウエスト/ヒップ/身長でファセット検索）。
- Vercel AI SDK の`streamUI`でAI自然言語検索（「○○な雰囲気の女優」）→ AI流入CVRは通常の42%高
- 「女優名×ジャンル」ロングテール約5万種を自動生成でSEO流入の柱
- shadcn/ui: タブ（デビュー作/最新作/シリーズ別/同事務所）+ 横スクロールカルーセル + サンプル動画ホバープレビュー

### 形態E: スワイプフィード（新規層獲得）

1画面1作品の縦スクロール。サンプル動画（size_476_306を3〜5秒ループ）を自動再生。
- 右スワイプ=お気に入り / 左スワイプ=スキップ / 上スワイプ=詳細
- スワイプUI採用でCV 3〜5倍の事例あり（Bijou Commerce）
- LocalStorageにスワイプ履歴保存し協調+コンテンツフィルタで再ランキング
- CTA: 画面下部「FANZAで見る」スティッキー固定（タップ44×44px以上）

### 形態F: 同人・電子書籍 シリーズ完走トラッカー

SeriesSearch + ItemListでシリーズ全作品を自動取得、既読/未読トラッキング + 新刊通知。
- シリーズ完走ニーズが高い同人フロアに特化
- 「全巻まとめ買いリンク」+ クーポン連動で高単価CV
- 新刊監視: Route Handler + Cloudflare Workers cron で自動更新

## ポイント付与サービスの選択肢（未決定）

独自ポイント運営は法務リスクが高いため、まずゲーミフィケーションから始め、規模が出てから移行を検討するのが現実解。

### 選択肢A: ゲーミフィケーション（推奨スタート地点）

金銭的価値を持たないため**前払式支払手段の登録不要**、法務リスクほぼゼロ。

| 要素 | 内容 |
|------|------|
| バッジ | 「初回訪問」「10作品お気に入り」「シリーズ完走」など |
| レベル | 訪問回数・アクション数に応じてライトユーザー→コレクターへ昇格 |
| ストリーク | 連続ログインで「今日もチェック」習慣化 |
| 称号表示 | プロフィールに「FANZA通」「同人マスター」など |

→ **ヘビーユーザー化の導線として機能**し、サイト回遊率・通知登録率の向上に直結。

### 選択肢B: 独自ポイント（将来検討）

ユーザーがサイト上のアクションでポイントを獲得し、サイト内特典と交換。

- **付与トリガー例**: 毎日ログイン / お気に入り登録 / アフィリンククリック後コールバック
- **交換先例**: プレミアム推薦機能のアンロック / 広告非表示期間 / 特別プレイリスト生成権
- **法務注意**: ポイントを自社サービス内の特典とのみ交換する場合、前払式支払手段に該当しないが、金銭や他サービスへの換金が発生すると資金決済法の規制対象になる。弁護士確認必須。

### 選択肢C: DMMポイント訴求（最も手軽）

自分でポイントシステムを運営しない。FANZAで購入するとDMMポイントが貯まることを**サイト上で訴求**するだけ。

- 「この作品はポイント還元○%」をカード上に表示
- 「DMMポイント○倍キャンペーン中」バナーを連携時に掲出
- 実装コストほぼゼロ。自分のリスクなし。

**推奨ロードマップ**: 立ち上げ期はC（DMMポイント訴求）+ A（ゲーミフィケーション）で開始し、月間UU 1万超えたタイミングでBへ移行を検討。

## 推奨技術スタック

| 項目 | 採用技術 |
|------|---------|
| フレームワーク | Next.js App Router + Server Components |
| レンダリング | ISR (revalidate 60〜3600秒)。上位1000商品は`generateStaticParams`、残りは`dynamicParams=true`でオンデマンドISR |
| ホスティング | **Cloudflare Pages** (opennextjs-cloudflare) + Cloudflare KV |
| DB | Supabase（お気に入り・スワイプ履歴・価格履歴・通知登録） |

| PWA | `manifest.json` + Service Worker（オフラインキャッシュ・プッシュ通知） |
| 検索 | MeiliSearch（日本語タイポ補正、self-host or Cloud） |
| AI機能 | Vercel AI SDK（自然言語→APIクエリ変換、streamUI） |
| UI | shadcn/ui（ボトムナビ・スワイプ・スケルトン含む） |

> VercelとNetlifyは規約上FANZA系広告掲載が事実上不可。

## 1人・自動化メインの運営設計

手動作業をゼロに近づけるための自動化ポイント。

| タスク | 自動化手段 |
|--------|-----------|
| 価格変動の検知・記録 | Cloudflare Workers cron（毎時）→ DMM API → KV/DB と価格比較 → 変化をキューに積む |
| ランキング・セール情報の更新 | Cloudflare Workers cron（毎時）→ DMM APIフェッチ → KV/DB更新 |
| ISRキャッシュ再検証 | `revalidate` + `revalidatePath` でAPIデータ変更時に自動再生成 |
| X 値下げアラート投稿 | Cloudflare Workers cron（1日3回）→ 値下げキューから取得 → X API v2 で投稿 |
| デプロイ | GitHub push → Cloudflare Pages CI/CD で自動デプロイ |
| 新刊・新作監視 | Workers cron でシリーズ/女優の最新作を定期チェック → 変化があれば通知キューに積む |

**手動でやること（最小限）**:
- DMM管理画面の「おすすめ商品&検索ワード」を月1確認してSEO記事の種にする
- 法務・規約変更のウォッチ（年数回）
- Cloudflare / Supabase の料金確認

## X（Twitter）自動投稿設計

### 概要

| 項目 | 内容 |
|------|------|
| 投稿内容 | 値下げアラート（お気に入り登録数が多い作品の値下がり優先） |
| 頻度 | 1日1〜3回・定時（例: 7時 / 12時 / 20時） |
| API プラン | **Free プラン**で十分（月1500ツイート制限、1日3回×30日=90投稿） |
| 画像添付 | Free プランはメディアアップロードAPIが使えないためテキストのみ（Basic $100/月で画像付きに拡張可） |

### X Developer Account 取得手順

1. [developer.x.com](https://developer.x.com) でアカウント作成（投稿用Xアカウントでログイン）
2. 用途説明（英語）を入力 → Free プランで申請  
   例: *"I run a Japanese affiliate site for FANZA and want to automatically post price drop alerts for products. Posts will include affiliate links and will be 1-3 times per day."*
3. アプリを作成し、以下の4つのキーを取得して Cloudflare の環境変数（`wrangler secret put`）に登録:
   - `X_API_KEY`
   - `X_API_SECRET`
   - `X_ACCESS_TOKEN`
   - `X_ACCESS_TOKEN_SECRET`

### 投稿アーキテクチャ

```
[Cloudflare Workers cron: 毎時]
  └─ DMM API で監視対象商品の現在価格を取得
  └─ Supabase の price_history テーブルと比較
  └─ 値下がり検知 → sale_queue テーブルに INSERT（重複防止フラグ付き）

[Cloudflare Workers cron: 7時/12時/20時]
  └─ sale_queue から未投稿の値下げを取得（お気に入り登録数降順）
  └─ ツイートテキストを組み立て
  └─ X API v2 POST /2/tweets で投稿
  └─ sale_queue の posted フラグを更新
```

### ツイートフォーマット（テキストのみ）

```
🔥 値下げ速報

「{作品タイトル（30文字以内に切る）}」
¥{定価} → ¥{現在価格}（{割引率}%OFF）

▶ {affiliateURL}

#FANZA #値下げ #セール
```

### 重複投稿・スパム対策

- 同一作品は72時間以内に再投稿しない（`last_posted_at` で制御）
- 割引率10%未満は投稿しない（小幅な変動ノイズを除去）
- 1回の cron 実行で最大3件まで投稿（連投によるスパム判定回避）

## CVR・CTR改善施策

- **CTA文言**: 「クリック」→「無料サンプルを見る」など動詞+ベネフィット化 → CTR 25〜60%改善
- **CTA配置**: ファーストビュー / 本文中盤 / 記事末尾の3箇所 + スマホは下部スティッキー固定
- **回遊最適化（商品ページ下部）**: 似た作品 / 同シリーズ / 同女優 / 同タグ / 同価格帯 / 最近人気 を大量配置
- **まとめ買い導線**: 「このシリーズ一括」「同女優5本」など → 客単価向上 + 高料率
- **ソーシャルプルーフ**: 「24時間以内に○件購入」は**実データ連動必須**（虚偽表示は景表法違反）
- **軽量反応機能**: 👍/💤/🔥/😭の1タップリアクション → ソーシャルプルーフ + レコメンド精度 + 回遊率向上

- **PR表記必須**: 景表法ステマ規制（2023年10月施行）により記事冒頭・リンク近辺に「PR」「広告」表記
- **年齢確認ゲート**: FANZAコンテンツは必須。`age_check_done=1` Cookie + `middleware.ts`で`/age-check`リダイレクト
- **画像・素材**: DMM公式提供素材のみ使用。改変（モザイク追加・スタンプ・黒塗り）禁止
- **自己アフィリエイト禁止**: DMM規約で明示的に禁止
- **禁止表現**: 「業界No.1」「絶対稼げる」など根拠なき優良/有利誤認表現
- **削除申請窓口**: 著作権者・肖像権者からの要請対応フォームを設置
- **ポイント運営時**: 換金・他サービスへの交換がある場合は資金決済法の前払式支払手段に該当する可能性あり。弁護士確認必須。
- **最低支払額**: 5,000円
