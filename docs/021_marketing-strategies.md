### 施策 1｜X 自動投稿 — セール・値下げ速報

**担当主導: B / 支援: C**  
**実装コスト: 小**（X APIキー設定済み）  
**優先度: 🔴 最高**

**B の主張:**
オタク会社員がXで最も反応するのは「今日限りのお得情報」。深夜0時にセールが切り替わるFANZAの特性を活かし、0:05に自動投稿するだけで「一番早い速報」ポジションが取れる。フォロワーゼロでも「FANZA セール」検索からの流入が期待できる。

**C の補足:**
投稿文にサイトURLを入れることで、クリックユーザーをそのままPWAインストール→通知登録のファネルに流し込む。1投稿が1件のリテンション獲得に直結する。

**A の懸念:**
アカウントが成熟するまでリーチが低い。補完としてブログや女優ページにXウィジェットを埋め込み相互補完する。

**結論:**
`workers/x-post.ts` + `wrangler.toml` cron（`5 0 * * *`）で即実装可能。X APIキーは `.env.local` 設定済み。WOW フォーマット（割引率・価格変化）で投稿。

```
🔥 今日のFANZAセール TOP3
1位【82%OFF】「{タイトル}」¥{定価}→¥{価格}
2位【75%OFF】「{タイトル}」...
詳しくはこちら → {/sale URL}
#FANZA #セール情報
```

### 施策 2｜女優ページ SEO 強化 — 「[女優名] おすすめ」ロングテール

**担当主導: A**  
**実装コスト: 中**  
**優先度: 🔴 高**

**A の主張:**
「[女優名] おすすめ 作品」「[女優名] FANZA 人気」は明確な購買意図クエリ。現在の女優ページは作品グリッドがメインで、テキストコンテンツが薄い。Googleが評価するための「文章」が足りない。具体的には以下を追加する。

- h2「{名前} の人気作品ランキング」（review 順TOP5）
- h2「{名前} の関連女優」（共演頻度から集計済みのデータを活用）
- description: 「{名前}のFANZA作品を全{N}本掲載。B{x} W{y} H{z}。人気ジャンルは{top3}。」

**B の補足:**
女優ページに「Xでシェア」ボタン（既に実装済み）があるので、SEOで流入 → シェア → 新規流入のサイクルが作れる。

**C の視点:**
女優ページに到達したユーザーが次に「推し登録 → 通知許可」するCTAを設ければLTVが跳ね上がる。今は通知ベルが目立たない位置にある。ヒーローセクション直下にCTAを置くべき。

**結論:**
女優ページの `generateMetadata` と本文に構造化テキストを追加。既存の `fetchActressList` / `fetchItemList` のデータを再利用するため追加APIコールなし。

### 施策 3｜PWA インストール × 推し通知 — 最強リテンション導線

**担当主導: C**  
**実装コスト: 中**  
**優先度: 🔴 高**

**C の主張:**
このサイトの最大の差別化はPWA + Web Pushによるプッシュ通知。しかし現状、通知許可率が低いと予想される理由がある。「通知してもいいですか？」という唐突なブラウザプロンプトは拒否率80%超。**事前の価値提示が必須**。

「推し女優を登録すると→新作が出たときスマホに通知が届く」をオンボーディング画面で図解付きで説明してから許可ボタンを押させる。Age Check 通過直後の最初の1画面が適切なタイミング。

**B の補足:**
「新作通知」は口コミになりやすい。「こんなアプリ入れたら推しの新作を見逃さなくなった」をXに書いてもらえるかが鍵。通知テキストを「{女優名}の新作が出ました🎉 今すぐチェック」という温かみのある文体にする。

**A の視点:**
「PWAインストール方法」のFAQページをサイト内に作ると「FANZA PWA」「FANZAアプリ iPhone」などの検索流入が狙える。

**結論:**
Age Check 後のファーストビューにPWAインストール訴求 + 推し女優登録フローを追加。通知許可は2ステップ（自前モーダル → ブラウザプロンプト）にして拒否率を下げる。

### 施策 4｜「今日のFANZA セール速報」デイリー SEO ページ

**担当主導: A / 支援: B**  
**実装コスト: 小**（既存 `/sale` ページの改善）  
**優先度: 🟡 中高**

**A の主張:**
「FANZA セール 今日」「FANZA 割引 今日」は毎日検索されるクエリ。現在の `/sale` ページタイトルは汎用的。これを「今日のFANZAセール・割引作品（2026年6月1日更新）」のように日付入りにするだけでフレッシュネスシグナルが上がりCTRが改善する。

ISRで1時間ごとに更新されているので「毎時更新」もfact。

**B の補足:**
タイトルを「2026年〇月〇日のFANZA割引」にすると、X投稿との合わせ技で「検索→SNS→検索」の循環が生まれる。

**C の懸念:**
毎日更新だとインデックスの「鮮度」が上がる反面、old URLがキャッシュされ続けるリスクがある。`/sale` の canonical を固定して日付はh1・descriptionのみに使う設計が安全。

**結論:**
`/sale` の `generateMetadata` で動的に今日の日付を入れる。実装コスト2時間以内。

### 施策 6｜ジャンル別 FAQ — AI Overview（旧SGE）への引用狙い

**担当主導: A**  
**実装コスト: 中**  
**優先度: 🟡 中**

**A の主張:**
Googleの AI Overview は FAQPage 構造化データから直接引用する。既に BreadcrumbList + FAQPage を実装済み。しかし現在のFAQ内容は汎用的すぎる。

例えば「NTR ジャンルとは」「素人 FANZAとは何か」「人妻 特典は」といった**具体的なQ&A**を各ジャンルページに追加すると、AI Overview の回答に採用される確率が上がる。採用されれば検索結果の目立つ位置に露出できる。

**B の補足:**
AI Overview に引用されるとURLがゼロクリック（読むだけ）になるリスクもあるが、ブランド認知には確実につながる。「FANZAピックス」という名前が AI 回答内に出れば指名検索が増える。

**C の視点:**
FAQ に「このサイトでできること（通知・スワイプ）」を含めると機能訴求にもなる。

**結論:**
ジャンルページの FAQPage を充実させる。ジャンル名に合わせた動的生成が可能（現在は汎用文のみ）。上位50ジャンルに絞って手動でQ&Aを強化する。

### 施策 8｜匿名コミュニティへのシード投稿

**担当主導: B**  
**実装コスト: 小**（手動）  
**優先度: 🟡 中**

**B の主張:**
ペルソナは5ch・2ch・Redditの日本語版に確実にいる。特に5chの「AV女優板」「無修正・AV関連板」は一次情報が飛び交う場所。「こういうツール使ってます」というナチュラルな形で紹介するだけで、数百〜数千の見込みユーザーへのリーチが得られる。

重要: **直接的な宣伝はNG**。「セールで90%OFFの作品をまとめて見てたら、こういうサイトで整理した」という体験談形式が自然かつ効果的。

**A の補足:**
投稿後に検索エンジンがそのスレをインデックスすることもある。5chスレは「FANZA セール 2026」などで検索されることがある。

**C の視点:**
投稿タイミングはセール開始直後（深夜0:10頃）が最も効果が高い。ユーザーが情報を探している瞬間と合致する。

**結論:**
月1〜2回、大型セール時に手動でシード投稿する。「宣伝ではなくユーザー体験の共有」というトーンを維持する。

### 施策 9｜Web Push リストビルド — 「セール通知」で訴求軸を広げる

**担当主導: C / 支援: B**  
**実装コスト: 小〜中**  
**優先度: 🔴 高**

**C の主張:**
現在の通知訴求は「推し女優の新作」のみ。これは既に好きな女優がいる既存ファン向け。「まだ推しが決まっていないユーザー」にとって登録動機が薄い。

**「今日のセール開始時に通知」** という訴求軸を追加すると登録率が劇的に上がる。「毎日深夜に最大90%OFFのセールが始まったら通知が来る」は明確なバリューだ。

さらに、通知一覧画面または登録フォームに「現在 {N}人が登録中」とDBから実数で表示するソーシャルプルーフを加えると、登録ためらいが減る。

**B の視点:**
通知で送るテキストのクオリティが口コミを左右する。「{女優名}の新作が出ました」より「{女優名}に待望の新作🎉 今だけ初日特典あり」のような熱量ある文体の方がシェアされやすい。

**A の補足:**
通知経由の流入は直帰率が低くSEOのエンゲージメントシグナルにもプラス。

**結論:**
通知登録フローに「推し女優通知 OR セール速報通知」を選べるオプションを追加。既存の Supabase `notification_subscriptions` テーブルに `notification_type` カラムを追加するだけで実現可能。

### 施策 10｜お気に入りランキング X シェア機能

**担当主導: B・C 共同**  
**実装コスト: 小**  
**優先度: 🟡 中**

私のFANZAお気に入り TOP3 🎯

1位「{タイトル}」★{評価}
2位「{タイトル}」★{評価}

**A の補足:**

# 第2部｜トップ専門家2名による施策レビューと追加提案

> 第1部の3名は「実務の担当者」。第2部は一段上のレイヤーから全体戦略・リスク・チャネル選定を見直すレビュー。**ここで第1部の施策のうち2つは「そのまま実行すると危険」と判定される。**

## レビュアー2名

| ロール | 経歴イメージ | 役割 |
|-------|------------|------|
| **D（事業戦略 / リスク管理）** | 上場企業のCMO・メディア事業の立ち上げ複数。プラットフォームポリシーと法規制に精通 | 「その施策、そもそも実行できるのか？潰されないか？」 |
| **E（アダルト垂直のグロース専門）** | アダルト動画・出会い系・ライブ配信のグロースを10年。一般向けノウハウが通用しない領域を熟知 | 「アダルトでは一般論のセオリーが逆効果になる。現場の正解はこれ」 |

## 総評：第1部の最大の盲点は「アダルトであること」

**D：** 第1部はよくできている。が、致命的な前提が抜けている。**この事業はアダルトコンテンツのアフィリエイトだ。** 一般サービスのマーケ施策をそのまま適用すると、主要プラットフォームのポリシー違反で**アカウント即凍結**になる施策が混ざっている。具体的には施策1（X自動投稿）と施策8（5chシード）。

**E：** 同感。私はアダルト領域を10年見てきたが、「一般向けに有効な施策ほどアダルトでは禁止されている」のが鉄則だ。GoogleのSafeSearch、Xのセンシティブメディア規制、Metaの全面禁止、メールのスパムフィルタ……一般チャネルは軒並みアダルトに敵対的。**チャネル選定そのものを設計し直す必要がある。**

**D：** つまり第1部は「施策の質」は高いが「実行可能性とチャネルリスクの査定」が甘い。第2部ではまずそこを潰す。

## 重大リスク指摘（第1部が見落としたもの）

### 🔴 リスク1：X（Twitter）のアダルトアフィリエイト凍結（施策1・5・10に影響）

**E：** Xはアダルト表現自体は許容しているが、**「アフィリエイトリンクを含むアダルト投稿の自動化」は最も凍結されやすいパターン**だ。特に：
- 新規アカウント＋外部リンク＋アダルト＋bot投稿 = 凍結フラグ4つ重なり
- `al.fanza.co.jp` のアフィリンクは過去に大量凍結された実績あり

**D：** 対策なしに施策1を `cron` で自動化すると、数日で凍結されて全工数が無駄になる。**実行前に必ず以下を設計する：**
1. アカウントを「センシティブメディア」設定にする（必須）
2. アフィリンクを**直貼りしない**。プロフィールのリンク or 自社サイト経由のワンクッションを挟む（`fanzapicks.com/sale` に飛ばし、そこからFANZAへ）
3. 投稿頻度を1日1〜2回に抑え、bot臭を消す（時刻をランダム化）
4. 画像はDMM公式サムネのみ（規約上の改変禁止も遵守）

**E：** むしろ**自社サイト経由に一本化する**のが正解。Xでは「セール情報＋自社サイトURL」だけ流す。アフィリンクはサイト内にだけ置く。これでXの凍結リスクとFANZAの自己アフィリ規約の両方をクリアできる。

### 🔴 リスク2：ステマ規制（景表法）抵触（施策8）

**D：** 2023年10月施行の**ステルスマーケティング規制**により、事業者が自社サービスを第三者を装って宣伝する行為は景表法違反（不当表示）になる。施策8の「5chに体験談を装って投稿」は**運営者本人がやれば明確に違法**。

**E：** これは絶対にやってはいけない。代わりに、**「PR」と明示したうえでの情報提供**ならOK。もしくは、ユーザーが自発的に書き込みたくなる仕掛け（施策10のシェア機能）に投資する方が、合法かつ持続的。施策8は**「自作自演」を完全に外し、純粋な広告出稿 or アフィリエイター募集に置き換える**べき。

### 🟡 リスク3：アダルトはGoogle検索で構造的に不利（施策2・4・6・7に影響）

**E：** SafeSearchはデフォルトON化が進み、アダルトサイトは一般クエリでほぼ表示されない。SEO施策の天井は一般ジャンルより**圧倒的に低い**ことを織り込むべき。

**D：** だからといってSEOが無駄ではない。**指名性の高いクエリ（「女優名」「作品名」「FANZA セール」）に絞れば**SafeSearchの影響下でも露出できる。汎用ビッグワードを狙わず、ロングテール×指名検索に集中する。第1部の施策2・4の方向性は正しいが、「ビッグワードSEO」への色気は捨てる。

## 追加提案（D・Eによる新規施策）

### 追加施策 11｜アダルト許容チャネルへの集中投下【E主導・最優先】

**実装コスト: 小〜中｜優先度: 🔴 最高**

**E：** 一般チャネル（X/TikTok/Instagram/Google）がアダルトに敵対的なら、**最初からアダルトを許容するチャネルに張る**のが垂直グロースの定石。具体的には：

- **DMM/FANZA エコシステム内**：FANZAの公式アフィリエイトの導線、関連サービスからの送客
- **アダルトまとめ系・アンテナサイト**：相互リンク・RSS配信で送客（アダルトアフィリの王道）
- **Telegram / Discord**：アダルトコンテンツ配信が許容され、プッシュ性も高い。「セール速報チャンネル」を作れば施策9と同じ効果を凍結リスクなしで実現
- **メールマガジン（自社）**：プラットフォーム非依存。一度取得すれば永続資産

**D：** これは戦略の根幹だ。第1部は「みんながいるX/Google」に固執していたが、**ペルソナ（オタク会社員）はTelegram/Discord/専用まとめサイトにも必ずいる**。むしろそこの方が濃い。チャネルを「敵対的だが大きい」から「友好的で濃い」へ振り直す。

### 追加施策 12｜効果測定・KPIダッシュボードの整備【D主導】

**実装コスト: 中｜優先度: 🔴 高**

**D：** 第1部には**測定の設計が完全に欠落**している。施策を10個並べても、どれが効いたか分からなければ改善できない。1人運営だからこそ、数字で「やめる施策」を即決できる体制が要る。最低限：

- 流入元別の計測（UTMパラメータ / `?ref=` の徹底）
- ファネル可視化：**訪問 → Age通過 → PWA導入 → 通知許可 → アフィリンククリック → 再訪**の各段階の数値
- 既存の Supabase に `events` テーブルを追加し、自前でファネル集計（GA4はアダルトで使えない可能性があるため自前計測が安全）

**E：** アダルトはGA4/Search Consoleが規約で使えない・凍結されるケースがある。**自前のイベント計測は必須インフラ**。これがないと全施策が博打になる。

### 追加施策 13｜「セール速報」をコンテンツ資産化（Telegram/メール × 自動化）【D・E共同】

**実装コスト: 中｜優先度: 🔴 高**

**E：** 施策1（X速報）・施策9（Webプッシュ）・追加施策11を統合する。**「FANZAセール速報」という1つのコンテンツ資産を、複数チャネルに同時自動配信**する。

- 1本の生成ロジック（既存の `fetchDailySaleItems` を流用）→ X / Web Push / Telegram / メール へファンアウト
- どこかのチャネルが凍結されても他で生き残る**分散配信アーキテクチャ**
- ペルソナが「速報の一次ソース」として認知すれば、それ自体がブランドになる

**D：** これが第1部の施策1・9・追加11を束ねる**上位戦略**だ。個別チャネルに一喜一憂せず、「速報コンテンツ」という資産を中心に据える。Cloudflare Workers cron で既に価格監視基盤があるので、配信先を増やすだけで実装できる。

### 追加施策 14｜既存ゲーミフィケーション（バッジ）をリテンション燃料に転用【E主導】

**実装コスト: 小｜優先度: 🟡 中**

**E：** このサイトには既にバッジ機能（`lib/badges.ts`）が実装済みなのに、第1部で一切触れられていない。**作ったのに使っていない資産は最ももったいない。** 「7日連続ログインバッジ」「コレクター50バッジ」は再訪の強力な動機。通知（施策9）と組み合わせて「あと1日でバッジ獲得」のリマインドを送れば、リテンションが跳ねる。

**D：** 新規施策を作る前に、**既存の埋もれた機能を起こす**のが1人運営では費用対効果が最も高い。実装ゼロで効果が出る。

## D・E 最終総括

**D：** 第1部の施策の質は高い。だが「アダルト × 1人運営」という制約下では、**勝負どころは"プラットフォームに依存しない自社チャネル（PWA・プッシュ・メール・Telegram）"の一点**に集約される。X やGoogleは「凍結・除外されても痛くない補助線」と位置づけ、本丸に資源を張るべきだ。

**E：** 同意。アダルトグロースの鉄則は3つ。**(1) 一般チャネルのセオリーを疑う、(2) アダルト許容チャネルに早く張る、(3) 自社リスト（通知・メール）を全力で太らせる。** 第1部はこの3つが弱かった。第2部で軌道修正できれば、1人運営でも十分戦える事業になる。

> **第2部の結論**: 着手順は **Phase 0（計測・バッジ）→ Phase 1（自社チャネル）→ Phase 2（友好チャネル）**。X自動投稿（施策1）は「自社サイト経由＋センシティブ設定＋頻度制御」を設計してから着手。5chシード（施策8）は**ステマ規制のため現状の形では実行しない**。
